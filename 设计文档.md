# FastRead 设计文档

## 1. 产品概述

FastRead 是一款常驻 Windows 系统托盘的阅读辅助工具。用户在任意支持文本复制的应用中选中文字后，可通过全局快捷键请求大模型进行“总结”或“详细解释”，并在轻量结果窗口中查看、复制结果。

产品目标：

- 不打断原有阅读流程，不要求把文本手动粘贴到聊天软件。
- 总结与解释使用不同快捷键、不同提示词和独立长度配置。
- 模型服务可配置，兼容 OpenAI Chat Completions API 及同协议服务。
- 本地安全保存 API Key，不记录用户选中的原文与模型回答。
- 对快捷键冲突、无选区、网络失败、响应格式错误提供明确反馈。

## 2. 使用场景

### 2.1 快速总结

1. 用户在浏览器、PDF 阅读器、Word、记事本等应用中选中一段文本。
2. 按下“总结”全局快捷键，默认 `Ctrl+Alt+S`。
3. FastRead 模拟复制选区，恢复原剪贴板内容，并请求模型。
4. 结果窗口显示结构化摘要，用户可复制或关闭。

### 2.2 详细解释

流程与总结相同，使用默认快捷键 `Ctrl+Alt+E`。提示词要求模型说明概念、上下文、逻辑关系与必要示例。

### 2.3 个性化配置

用户可从托盘菜单打开设置，调整：

- API 地址、模型名称与 API Key；
- 总结、解释分别允许的最大输出字数；
- 输出语言（简体中文或 English）；
- 两个功能的全局快捷键；
- 请求超时；
- 是否随 Windows 启动。

## 3. 功能范围

### 3.1 首个版本包含

- Windows 全局快捷键注册与冲突检测；
- 通过复制命令获取当前选区，尽量恢复原剪贴板；
- 总结、详细解释两种任务；
- 可配置兼容 OpenAI 的模型服务；
- 流式展示生成结果；
- 取消进行中的请求；
- 结果复制、错误提示、系统托盘；
- 设置校验、连接测试；
- API Key 使用 Windows 凭据管理器保存；
- 单实例运行与随系统启动。

### 3.2 暂不包含

- OCR 或屏幕截图取词；
- 不允许复制内容的 DRM 阅读器适配；
- 多轮对话、云端历史记录；
- macOS、Linux 与移动端；
- OpenAI Responses API 等非 Chat Completions 协议的专用适配。

## 4. 交互设计

### 4.1 系统托盘

程序启动后常驻托盘。菜单包含：

- “设置”；
- “使用说明”；
- “退出”。

双击托盘图标打开设置。

### 4.2 结果窗口

结果窗口位于当前鼠标所在屏幕中央，并保持在最前方。窗口包含：

- 任务标题与状态；
- 只读的结果文本区域；
- “复制结果”“取消/关闭”按钮；
- 请求阶段可见的进度提示。

窗口可调整大小。新请求到达时复用窗口并取消旧请求，避免回答串线。

### 4.3 设置窗口

设置按“模型服务、输出、快捷键、常规”分组。保存前执行本地校验：

- API 地址必须为 `http` 或 `https` 绝对地址；
- 模型名称不能为空；
- 输出字数必须在 50～20,000 之间；
- 请求超时必须在 5～300 秒之间；
- 两个快捷键不能相同，且必须包含至少一个修饰键。

“测试连接”使用当前表单值发送极短请求，不必先保存。

## 5. 技术设计

### 5.1 技术栈

- .NET 8；
- Windows Forms；
- `HttpClient` 调用模型服务；
- Win32 `RegisterHotKey` 实现全局快捷键；
- Win32 `SendInput` 发送 `Ctrl+C`；
- Windows Credential Manager 保存 API Key；
- `%LOCALAPPDATA%/FastRead/settings.json` 保存非敏感配置。

选择 Windows Forms 是因为首版高度依赖 Windows 的托盘、全局热键和剪贴板能力，且可避免引入 Electron 等大型运行时。

### 5.2 模块划分

| 模块 | 职责 |
| --- | --- |
| `Program` | 单实例控制、异常兜底、启动应用上下文 |
| `FastReadApplicationContext` | 托盘生命周期、热键调度、结果窗口协调 |
| `HotkeyWindow` | 注册/注销全局热键并接收 `WM_HOTKEY` |
| `SelectionCaptureService` | 备份剪贴板、发送复制、等待新文本、恢复剪贴板 |
| `LlmClient` | 构造请求、解析 SSE 流与普通 JSON 响应 |
| `PromptFactory` | 生成总结/解释提示词，约束目标输出字数 |
| `SettingsStore` | 配置持久化、默认值与校验 |
| `CredentialStore` | 通过 Windows 凭据管理器读写 API Key |
| `SettingsForm` | 编辑配置、测试连接 |
| `ResultForm` | 显示状态、流式内容、复制与取消 |

### 5.3 数据流

```text
用户选中文字
    ↓ 全局快捷键
HotkeyWindow
    ↓
SelectionCaptureService ── 备份/恢复 ── Windows Clipboard
    ↓ 已选文本
PromptFactory → LlmClient → 可配置模型 API
                       ↓ SSE/JSON
                    ResultForm
```

### 5.4 选区读取策略

Windows 没有跨应用读取“当前选中文本”的统一 API，因此使用通用复制策略：

1. 保存剪贴板中可序列化的数据对象；
2. 写入内部标记文本，确保能检测到复制是否真正发生；
3. 向前台应用发送 `Ctrl+C`；
4. 最多等待约 1.2 秒，轮询剪贴板文本是否改变；
5. 读取新文本；
6. 尽力恢复原剪贴板。

限制：目标应用必须允许复制；某些管理员权限应用无法接收普通权限程序的输入；复杂或超大剪贴板对象可能无法完整恢复。失败时不调用模型并显示操作建议。

### 5.5 模型接口

请求发送至用户配置的 Chat Completions 地址，例如：

```json
{
  "model": "MiniMax-M3",
  "messages": [
    { "role": "system", "content": "..." },
    { "role": "user", "content": "..." }
  ],
  "temperature": 0.2,
  "stream": true
}
```

请求头使用 `Authorization: Bearer <API Key>`。客户端优先解析 `text/event-stream`，同时兼容服务端忽略 `stream` 后返回的普通 JSON。

“输出长度”以目标字符数表达，并同时转换为 `max_tokens` 的保守估算值；提示词明确要求在指定字符数内完成回答。由于不同模型的分词和遵循能力不同，字符数是目标上限而非绝对保证，界面最终会在目标值附近安全截断并给出提示。

### 5.6 配置模型

非敏感配置示例：

```json
{
  "apiUrl": "https://api.minimaxi.com/v1/chat/completions",
  "model": "MiniMax-M3",
  "outputLanguage": "zh-CN",
  "summaryMaxChars": 500,
  "explanationMaxChars": 1500,
  "summaryHotkey": "Ctrl+Alt+S",
  "explanationHotkey": "Ctrl+Alt+E",
  "requestTimeoutSeconds": 60,
  "startWithWindows": false
}
```

API Key 单独保存在当前 Windows 用户的凭据管理器中，目标名为 `FastRead/ApiKey`。

## 6. 安全与隐私

- 只有用户主动按下快捷键后才读取复制得到的文本；
- 原文和回答只驻留内存，不写日志、不建立历史记录；
- API Key 不出现在 `settings.json`、异常消息或界面明文回显中；
- 默认只接受 HTTPS；为兼容本机代理允许 `http://localhost` 与回环地址；
- 请求取消时立即停止读取网络流；
- 错误信息过滤 API Key 和过长的服务端响应。

## 7. 异常处理

| 情况 | 行为 |
| --- | --- |
| 未配置 API Key | 打开设置并提示配置 |
| 无选中文本/复制超时 | 提示重新选择，保留剪贴板 |
| 热键注册冲突 | 保存失败，指出冲突的快捷键 |
| 401/403 | 提示检查 API Key 与服务权限 |
| 429 | 提示请求过快或额度不足 |
| 其他非 2xx | 显示状态码和经过截断的错误摘要 |
| 网络超时 | 提示调整超时或检查网络 |
| SSE 中断 | 保留已生成部分并提示连接中断 |

## 8. 验收标准

- 在记事本、主流浏览器中选中文本后，两种默认快捷键均可触发对应任务；
- 原剪贴板中的普通文本能在取词后恢复；
- 设置重启后仍然生效，API Key 不出现在配置文件；
- 支持流式与非流式兼容接口；
- 请求可以取消，重复触发不会混合两次回答；
- 无 API Key、无选区、超时、认证失败均有可理解的中文提示；
- `dotnet build` 和自动化测试通过；
- README 覆盖构建、配置、使用、限制与常见问题。
